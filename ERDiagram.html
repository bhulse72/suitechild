ER Diagram

Variable Meanings

Status Number:

0. Order needs to be placed/reviewed by buying team

1. Order under review - In progress or Priority

2. Something is preventing the order from being confirmed

3. Vendor has confirmed the order

- flags the inventory team to get involved

4. Order is on backorder

5. Order physically arrives at one of the stores (ELK or KEA)

6. There is a discrepancy in the order

7. Order is confirmed and received with no outstanding discrepancies

8. Inventory team has reconciled the order – everything matches in POS

Status Codes:

Purple: high priority

Orange: needs purchaser attention

Pink: high priority creation : still need a digital PO (internal ID#)

Yellow: needs CS approval

Red: Needs Inventory Ops Team attention

Green: ready to receive

Team Roles

Admin:

assumes all responsibilities of other teams ; manages other teams ; an see all data

General Employee:

Notifies the Buying team that they are running low on a product or there is high demand

Buying Team:

Gets a notification that order needs to be placed ; enters a new order - brand name and statues = 0 ONLY ; Reviews all status 0 line items ; if it needs ordered, based on vendor relationships, budget, and timing → set the status to “1” ; Contacts vendor ; and places the order; fills in order details, prices, notes, ship date, discounts, collections, etc ; Once vendor confirms the order, buying team changes the status from a “1” or “2” to a “3”

Inventory Team:

gets involved once the order status is set to a “3” ; Reviews all items with the “3” status ; creates the product digitally in POS ; creates purchase number (Heartland PO#) ; Updates code (pink or green) ; Once order is “5” and green → begin receiving process ; flags and tracks backorders ; reconciles orders ; resolve discrepancies

Sales Team:

Marks orders as a “5” when order arrives in store ; Notices and marks discrepancies (“6”) ; Marks orders as “7” when correct
